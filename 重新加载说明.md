# 插件重新加载说明

## 🔧 问题：插件功能不生效

当您修改了插件代码或遇到功能异常时，需要重新加载插件。

### 🎯 重新加载步骤

1. **打开扩展管理页面**
   - 在浏览器地址栏输入 `chrome://extensions/`
   - 或通过菜单：更多工具 → 扩展程序

2. **找到"复制记录板"插件**
   - 在扩展列表中找到本插件

3. **点击刷新按钮**
   - 点击插件卡片上的刷新图标 🔄

4. **刷新当前网页**
   - 按 `F5` 或 `Ctrl+R` (Windows/Linux)
   - 按 `Cmd+R` (Mac)

### ✅ 何时需要重新加载

以下情况需要重新加载插件：

- ✏️ 修改了插件代码文件
- 🐛 插件功能突然失效
- ⚠️ 看到 "Extension context invalidated" 错误
- 🔄 更新了插件版本

### 🚀 验证插件是否正常工作

1. **测试文本复制**
   - 在任意网页选中文字
   - 按 `Ctrl+C` 复制
   - 打开插件弹窗查看是否有新记录

2. **查看控制台日志**
   - 按 `F12` 打开开发者工具
   - 切换到 Console 标签
   - 复制文本时应该看到 "✅ 自动检测到文本复制" 的日志

### 📝 错误处理改进

当前版本已经改进了错误处理：

- **智能降级**：API失败时自动使用备选方案
- **非阻塞保存**：保存失败不影响复制功能
- **详细日志**：提供完整的调试信息

### 🔴 如果重新加载后仍然不工作

1. **完全删除并重新安装**
   - 在扩展管理页面点击"移除"
   - 重新点击"加载已解压的扩展程序"
   - 选择插件文件夹

2. **检查权限**
   - 确保插件已启用
   - 确认已授予剪贴板读取权限

3. **查看错误信息**
   - 打开开发者工具 (F12)
   - 查看 Console 中的错误信息
   - 如有错误，可以联系开发者

---

**提示**: 每次修改代码后都需要重新加载插件才能生效！
